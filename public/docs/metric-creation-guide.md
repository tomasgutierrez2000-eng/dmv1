# Metric Creation Guide

This guide explains how to create new metrics for the banking data model using the Excel template and Python calculators.

---

## Quick Start

1. **Download the template** from `/metrics/library/upload` (click "Download Template")
2. **Fill it out** using the instructions below (or paste this guide + the DD Reference sheet into your AI)
3. **Write a Python calculator** (Full or Simple mode) in `scripts/calc_engine/calculators/`
4. **Upload** the filled template at `/metrics/library/upload`
5. **Review** the validation report and fix any issues

---

## Data Model Overview

The model has three layers:

| Layer | Purpose | Examples |
|-------|---------|---------|
| **L1** | Reference/dimension data (rarely changes) | `counterparty`, `facility_master`, `currency_dim` |
| **L2** | Atomic/raw data (snapshots, events) | `facility_exposure_snapshot`, `credit_event`, `position` |
| **L3** | Derived/calculated data (metrics, aggregations) | `exposure_metric_cube`, `facility_financial_calc` |

**Rollup hierarchy:** Facility -> Counterparty -> Desk (L3) -> Portfolio (L2) -> Business Segment (L1)

## Filling the Template

### Metrics Sheet

Each row defines one metric. Required fields:

| Field | What to fill | Example |
|-------|-------------|---------|
| `metric_id` | `DOMAIN-NNN` format. Domain prefix = EXP, RSK, PRC, PROF, CAP, AMD, REF | `EXP-050` |
| `name` | Human-readable name | `Loan-to-Value Ratio` |
| `domain` | One of: exposure, risk, pricing, profitability, capital, amendments, reference | `exposure` |
| `abbreviation` | Short name for UI display (2-8 chars) | `LTV` |
| `definition` | Business definition in 1-3 sentences | `Ratio of outstanding loan balance to the appraised value of collateral` |
| `generic_formula` | Human-readable formula | `drawn_amount / collateral_value * 100` |
| `unit_type` | CURRENCY, PERCENTAGE, RATIO, COUNT, RATE, BPS, DAYS, INDEX | `PERCENTAGE` |
| `direction` | HIGHER_BETTER, LOWER_BETTER, NEUTRAL | `LOWER_BETTER` |
| `metric_class` | SOURCED (read directly), CALCULATED (derived), HYBRID (mix) | `CALCULATED` |
| `calculator_mode` | `full` (BaseCalculator subclass) or `simple` (facility_level function only) | `simple` |

Optional fields:

| Field | What to fill |
|-------|-------------|
| `insight` | Why this metric matters to the business |
| `rollup_strategy` | How to aggregate: `direct-sum`, `sum-ratio`, `weighted-avg` |

### SourceFields Sheet

Each row maps a metric to a source field in the data dictionary.

| Field | What to fill | Example |
|-------|-------------|---------|
| `metric_id` | Must match a metric_id from the Metrics sheet | `EXP-050` |
| `layer` | L1, L2, or L3 | `L2` |
| `table` | Table name from the DD Reference sheet | `facility_exposure_snapshot` |
| `field` | Field name from the DD Reference sheet | `drawn_amount` |
| `role` | MEASURE (value used in formula), DIMENSION (grouping), FILTER (date/flag), JOIN_KEY (FK) | `MEASURE` |
| `description` | What this field represents in the formula | `Outstanding drawn balance` |

**Important:** Use exact table and field names from the DD Reference sheet. The validator will flag mismatches.

## Python Calculator Modes

The calculation engine is Python-based. Every metric needs a Python calculator file in `scripts/calc_engine/calculators/`. There are two modes:

### Full Mode (BaseCalculator subclass)

Use Full mode when you need custom rollup logic at each level (counterparty, desk), or when the rollup is not a simple aggregation of facility-level results.

A Full mode calculator is a class that extends `BaseCalculator` with:
- **Attributes:** `metric_id`, `catalogue_id`, `name`
- **Methods:**
  - `facility_level(self, loader, as_of_date) -> pd.DataFrame`
  - `counterparty_level(self, loader, as_of_date) -> pd.DataFrame`
  - `desk_level(self, loader, as_of_date) -> pd.DataFrame`

Each method must return a DataFrame. The `@register` decorator from `..registry` auto-registers the calculator with the engine.

### Simple Mode (facility_level function only)

Use Simple mode when the metric only needs facility-level logic and the counterparty/desk rollups can be auto-generated by summing or averaging the facility results.

A Simple mode calculator is just a standalone `facility_level(loader, as_of_date)` function. The system auto-generates counterparty and desk rollups from the facility-level result based on the `rollup_strategy`.

## DataLoader API

The `DataLoader` provides access to all tables in the data model. Use it inside your calculator functions.

### Loading Tables

```python
df = loader.load_table("L2", "facility_exposure_snapshot")
```

- First argument: layer (`"L1"`, `"L2"`, or `"L3"`)
- Second argument: table name (exact name from the data dictionary)
- Returns: a `pandas.DataFrame` with all rows from that table

### Filtering by Date

```python
from .base import filter_by_date

fes = loader.load_table("L2", "facility_exposure_snapshot")
fes = filter_by_date(fes, as_of_date)
```

The `filter_by_date(df, as_of_date)` helper filters a DataFrame to rows where the `as_of_date` column matches the given date.

### Common Tables to Load

| Table | Layer | Key Fields | Use For |
|-------|-------|------------|---------|
| `facility_exposure_snapshot` | L2 | drawn_amount, committed_facility_amt, gross_exposure_usd | Exposure metrics |
| `facility_master` | L2 | facility_id, counterparty_id, lob_segment_id | Joining to hierarchy |
| `counterparty` | L1 | counterparty_id, legal_name, risk_rating_code | Counterparty info |
| `collateral_snapshot` | L2 | current_valuation_usd, facility_id | Collateral/LTV metrics |
| `facility_financial_calc` | L3 | dscr_value, ltv_pct, net_income_amt | Pre-calculated ratios |
| `credit_event` | L2 | event_type_code, severity_code | Risk event metrics |
| `facility_risk_snapshot` | L2 | risk_rating_code, pd_value | Risk rating metrics |
| `position` | L2 | balance_amount, product_code | Position-level data |
| `enterprise_business_taxonomy` | L1 | managed_segment_id, parent_segment_id | Desk/Portfolio/LOB rollup |

## Full Calculator Example

This example shows a complete BaseCalculator subclass for committed exposure:

```python
import pandas as pd
from ..registry import register
from .base import BaseCalculator, filter_by_date


@register
class CommittedExposureCalculator(BaseCalculator):
    metric_id = "EXP-001"
    catalogue_id = "MET-001"
    name = "Committed Exposure"

    def facility_level(self, loader, as_of_date):
        fes = loader.load_table("L2", "facility_exposure_snapshot")
        fes = filter_by_date(fes, as_of_date)
        return fes[["facility_id", "committed_facility_amt"]].rename(
            columns={"committed_facility_amt": "metric_value"}
        )

    def counterparty_level(self, loader, as_of_date):
        fes = loader.load_table("L2", "facility_exposure_snapshot")
        fes = filter_by_date(fes, as_of_date)
        fm = loader.load_table("L1", "facility_master")
        merged = fes.merge(fm[["facility_id", "counterparty_id"]], on="facility_id")
        return (
            merged.groupby("counterparty_id")["committed_facility_amt"]
            .sum()
            .reset_index()
            .rename(columns={"committed_facility_amt": "metric_value"})
        )

    def desk_level(self, loader, as_of_date):
        fes = loader.load_table("L2", "facility_exposure_snapshot")
        fes = filter_by_date(fes, as_of_date)
        fm = loader.load_table("L1", "facility_master")
        merged = fes.merge(fm[["facility_id", "lob_segment_id"]], on="facility_id")
        return (
            merged.groupby("lob_segment_id")["committed_facility_amt"]
            .sum()
            .reset_index()
            .rename(columns={"committed_facility_amt": "metric_value"})
        )
```

## Simple Calculator Example

This example shows a facility_level function for drawn exposure. The system auto-generates counterparty and desk rollups.

```python
import pandas as pd
from ..registry import register
from .base import BaseCalculator, filter_by_date


def facility_level(loader, as_of_date):
    fes = loader.load_table("L2", "facility_exposure_snapshot")
    fes = filter_by_date(fes, as_of_date)
    return fes[["facility_id", "drawn_amount"]].rename(
        columns={"drawn_amount": "metric_value"}
    )
```

The function must return a DataFrame with at least `facility_id` and `metric_value` columns. The rollup strategy (defined in the Metrics sheet) determines how counterparty and desk levels are aggregated.

---

## Rollup Strategies

| Strategy | When to use | How rollup works |
|----------|------------|-----------------|
| `direct-sum` | Additive metrics (amounts, counts) | SUM values at each level |
| `sum-ratio` | Ratio metrics (LTV, coverage) | SUM(numerator) / SUM(denominator) at each level |
| `weighted-avg` | Weighted metrics (rates, scores) | Weighted average by exposure at each level |
